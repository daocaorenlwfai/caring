# Caring 产品需求文档 (PRD)

## 1. 产品定位

一款温暖的**亲友关怀提醒工具**，帮助用户记录亲友的重要事件，并在合适的时机提醒用户去关心亲友的近况。

核心价值：不让忙碌成为忘记关心亲友的借口。

---

## 2. 目标用户

- **主要用户**：25-40岁女性
- **使用场景**：记住亲友的重要事件（生病、怀孕、工作变动等），定时提醒关心
- **核心需求**：记录 + 提醒，不需要复杂的社交功能

---

## 3. 核心功能列表

| 功能模块 | 功能点 | 优先级 |
|---------|-------|-------|
| 亲友管理 | 添加/编辑/删除亲友 | P0 |
| 亲友管理 | 亲友分组（家人/朋友/同事等） | P1 |

| 事件记录 | 事件详情记录 | P0 |
| 提醒设置 | 自定义提醒间隔（3天/5天/7天/14天/30天等） | P0 |
| 提醒设置 | 提醒方式（微信服务号推送） | P0 |
| 通知中心 | 提醒通知 | P0 |
| 通知中心 | 记录已关心（稍后提醒） | P0 |
| 数据统计 | 累计关心次数 | P2 |

## 5. 数据模型

### 5.1 亲友表 (Contacts)

| 字段 | 类型 | 说明 |
|-----|-----|-----|
| id | int | 主键 |
| name | string | 亲友名称 |
| relation | string | 关系（爸爸、妈妈、公公、婆婆、朋友等） |
| group | string | 分组（家人/朋友/同事） |
| remark | string | 备注 |
| avatar | string | 头像URL |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### 5.2 事件表 (Events)

| 字段 | 类型 | 说明 |
|-----|-----|-----|
| id | int | 主键 |
| contact_id | int | 关联亲友ID |
| event_type | string | 事件类型 |
| event_title | string | 事件标题 |
| event_desc | string | 事件描述 |
| event_date | date | 事件日期 |
| created_at | datetime | 创建时间 |

### 5.3 提醒表 (Reminders)

| 字段 |类型 | 说明 |
|-----|-----|-----|
| id | int | 主键 |
| contact_id | int | 关联亲友ID |
| event_id | int | 关联事件ID |
| interval_days | int | 提醒间隔（天） |
| last_reminded_at | datetime | 上次提醒时间 |
| next_remind_at | datetime | 下次提醒时间 |
| is_active | bool | 是否启用 |
| created_at | datetime | 创建时间 |

## 6. MVP 优先级

### P0 - 必须实现

1. 添加亲友（姓名+关系）
2. 为亲友添加事件（类型+描述+日期）
3. 设置提醒间隔（固定选项：3/5/7/14/30天）
4. 微信服务号推送提醒
5. 今日提醒首页

### P1 - 应该实现

6. 亲友分组
7. 稍后提醒（1小时后再次提醒）
8. 亲友详情页展示所有事件

### P2 - 可以不做

- 数据统计
- 事件模板自定义
- 批量操作

---

## 7. 技术方案

### 方案：微信小程序云开发

| 组件 | 技术 |
|-----|-----|
| 前端 | 微信小程序原生开发 |
| 后端 | 微信小程序云开发（云函数+云数据库） |
| 存储 | 云数据库 |
| 推送 | 微信服务号模板消息 |

### 云数据库集合

- `contacts` - 亲友
- `events` - 事件
- `reminders` - 提醒
- `users` - 用户信息

### 提醒实现

- 使用云函数定时触发（每天8:00检查提醒）
- 通过服务号模板消息推送提醒
